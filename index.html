Что такое КТР и ИЛ

КТР (Коэффициент территориального распределения) — персональный коэффициент для каждого артикула. Зависит от его доли локальных заказов: больше локальных = ниже КТР = дешевле доставка этого артикула. У каждого артикула свой КТР — общего «КТР по кабинету» как отдельного понятия у WB нет.

ИЛ (Индекс локализации) — это и есть средневзвешенный КТР по всем заказам кабинета за 13 недель. Поэтому на верхней панели нет отдельной общей карточки «КТР» — карточка «КТР среднее» это и есть ИЛ, та же цифра. КТР по каждому конкретному артикулу смотри на вкладке «По артикулам».

Стоимость логистики = Тариф × Коэф.логистики × ИЛ + Цена × ИРП
ИЛ = Σ(КТР_артикула × Заказов_артикула) / Σ(Заказов)
ИРП = 0 при доле локализации ≥ 60%

Как считается локальный заказ

Заказ локальный если склад отправки и покупатель в одной зоне. По официальным правилам WB объединены в одну зону: ЮФО + СКФО и СФО + ДВФО.

Локальный = зона(ФО склада) == зона(ФО покупателя)
Доля локализации = Локальных / Заказов РФ+Казахстан × 100%

ФО покупателя ← поле oblastOkrugName (WB API)
ФО склада ← поле warehouseName → справочник ниже

Казахстан учитывается как локализуемая территория наравне с ФО РФ — так делает и сам WB при расчёте ИЛ.

Остальной СНГ исключён из расчёта (Армения, Беларусь, Киргизия, Узбекистан и др.) — не входит ни в числитель, ни в знаменатель доли локализации, как в официальной методике WB.

Источник данных: statistics-api.wildberries.ru/api/v1/supplier/orders?dateFrom=...&flag=0

flag=0 = фильтр по lastChangeDate, возвращает все актуальные заказы. Отменённые (isCancel:true) исключены. Лимит WB: 90 дней.

Индекс распределения продаж (ИРП)

Введён с 23 марта 2026 года. Считается по той же логике, что и ИЛ — на основе доли локализации каждого артикула — но по собственной таблице коэффициентов КРП (коэффициент распределения продаж), и применяется не к литражу, а к цене товара до скидки WB.

ИРП = Σ(КРП_артикула × Заказов_артикула) / Σ(Всех заказов)

При доле локализации ≥ 60% → КРП = 0% (ИРП не влияет на цену)
При доле локализации < 60% → КРП от 2.00% до 2.50%

ИРП в этом дашборде считается так же, как ИЛ — по заказам из /api/v1/supplier/orders. WB официально исключает из ИРП ряд категорий (КГТ+, СГТ, КБТ, заказы FBS) — эти исключения здесь не применяются, так как у нас нет признака модели продаж и габаритов товара в этом эндпоинте.

Возможные причины расхождения с кабинетом WB

  • WB официально предупреждает: «Общий % локальных заказов» (раздел География заказов) и «Индекс локализации» (раздел Тарифы складов) — это разные показатели с разной методикой, их значения у самого WB не совпадают друг с другом.
  • Часовые пояса: API отдаёт даты в UTC, кабинет WB показывает время продавца — на стыке суток заказ может попасть в соседние сутки.
  • Период ИЛ/ИРП у WB — строго 13 недель, скользящее окно с обновлением по понедельникам. Для точного сравнения выставляй точно такой же период.
  • Крупногабаритные и сверхгабаритные товары (КГТ+, СГТ, КБТ) WB исключает из расчёта ИЛ/ИРП полностью — в этом дашборде такого фильтра пока нет.
  • Минимальный порог в 1000 заказов за 13 недель — на меньших объёмах официальный ИЛ WB может быть нерепрезентативным.
  • Команда WB API на форуме разработчиков подтвердила известные случаи расхождения количества заказов между /api/v1/supplier/orders и интерфейсом личного кабинета.
